AWS S3: creating bucket policy

AWS S3: creating bucket policy

November 10, 2019 Marcelo Andrade

4 minute read

  • Twitter

  • Google+

  • Facebook

  • Reddit

  • LinkedIn

  • StumbleUpon

  • Pinterest

  • Email

Task at hand: create a bucket policy that makes an entire bucket public.

Resources used: AWS S3, custom domain registered.

So I created a website and hosted it on AWS S3. It worked just as advertised, which is nice, even though it could use a ‘press one button and be done with it’ just as their one-click-shop.

As noticed, I had to Grant public read access to all objects on upload. So if I mess up an object, my website can break because of my lack of attention.

There is a way to apply a global Public policy to the bucket. It’s a little intimidating, but let’s do it.

Creating a bucket policy

Everything just worked, and that’s nice. But there’s a catch: it only worked because I have, on the second screen of the upload operation, selected the Grant public read access to this object. You have to set this parameter on every single object you upload, or it won’t show up on the website.

Even though that’s not exactly a problem per se, there’s a smarter way to do this. Since this bucket will be solely used as a website hosting repository, I can create an AWS Policy that will apply to every object turning them public. Let’s see how to do this.

On the bucket screen, now it’s the time to click on the third tab, Permissions.

Here we can see that Block Public access is already Off. But now we’re after the Bucket Policy tab.

On Bucket Policy, a very empty numbered text editor waits for you to type a JSON object that describes a valid AWS policy. You’re joking, right? Not really. Fortunately the’res this subtle Policy Generator button at the bottom of the page.

Before clicking on the Policy Generator button, let’s get smart and copy/paste this weird string next to Bucket policy editor ARN (arn:aws:s3:::yellatcloud.devsres.com). It’ll save us to have to open another window.

The Policy Generator is a lot more friendly than the empty JSON text editor. It’s not that smart, though. It could have noticed I came from S3 (also, it could have noticed I came from bucket arn:aws:s3:::yellatcloud.devsres.com from S3, but whatever!). But hey, let’s not yell at the Cloud.

  • Step 1: Select policy type:
    • Well, it’s a S3 Bucket Policy, of course.
  • Step 2: Add Statement(s)
    • Effect: Allow, as we want to give public access to the Bucket.
    • Principal: Here he wants to know to whom this policy will apply (as in clients). Since we want it to be available to everyone, the answer here is: * .
    • AWS Service: As I picked S3 Bucket Policy, it won’t allow me to change anything here.
    • Actions: We want public access, but not public everything! Here we pick Get Object.
    • Amazon Resource Name (ARN): ah, the explanation of what this cryptic ARN thing is. Nicely thought of them. Now we paste what we copied, and add a /* at the end to specify that it applies to everything in the bucket: arn:aws:s3:::yellatcloud.devsres.com/* . Dont forget to press the AWS Store themed Add Statement or you’ll lose all work before next step.
  • Step 3: Generate Policy
    • Just click the yellow Generate Policy button.

The Policy Generator won’t drop the JSON code inside the window where you need it to be, you have to copy and paste the JSON to the window you were previously.

Back to the Bucket policy editor, drop the JSON bomb:

{
  "Id": "Policy1573369582399",
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "Stmt1573369528972",
      "Action": [
        "s3:GetObject"
      ],
      "Effect": "Allow",
      "Resource": "arn:aws:s3:::yellatcloud.devsres.com/*",
      "Principal": "*"
    }
  ]
}

Press Save.

Notice the new orange decorator that says This bucket has public access. It’s an nice warning, but that was exactly what we were expecting!

So, from now, every file uploaded will automatically have public acess.

Key notes

  • AWS Policy generator isn’t integrated at all to anything, it is there to make noobs like myself comfortable to write a couple policies.

Reference: AWS Policy Generator.
