AWS with Terraform: why can't I resolve my EKS node's hostname?

AWS and Terraform'ing for noobs

April 19, 2020 Marcelo Andrade

4 minute read

  • Twitter

  • Google+

  • Facebook

Note for Pros: Terraform noob tip. It’s not that relevant.

Preamble

I had a turbulent mission: learn both Terraform and AWS from scratch without ever using it before, building a full EKS cluster as IaaC. I also had two weeks.

So, as expected, it was a hurricane of copy and paste from examples in the Internet - mainly official Terraform documentation - and methodology was “if it’s created, it’s fine!".

A month after, it was time to actually read what was happening. So you start noticing where you failed without even knowing,

The matter at hand

I was finally done, and all that was missing on my fantabulous fully automated Terraformed EKS cluster was to install the Metrics Server, which was trivial enough, right? I’ve RTFM this time, launched the yaml and it’s up.

An hour later, I decided to throw some HPAs to test it. It was a no go.

Well, maybe you should check Pod’s logs before thinking they’re working just because they’re up!

E0903  1 manager.go:102] unable to fully collect metrics: [unable to fully scrape metrics from source kubelet_summary:<hostname>: unable to fetch metrics from Kubelet <hostname> (<hostname>): Get https://<hostname>:10250/stats/summary/: dial tcp: lookup <hostname> on 10.96.0.10:53: no such host

So it seems my AWS nodes’ hostnames aren’t “resolvable” by AWS DNS.

I did what you’d expect a seasoned, experienced sysop to do: I google’d this problem, and I google’d hard. For some reason, had absolutely no hints on what was wrong.

I was even directed to this massive wall of text to support a very complex scenario that had nothing to do with my problem - and which I read throughly because I failed to read the “Update - December 2019” part that says that guide wasn’t necessary anymore because they changed something.

Since I couldn’t find anyone with this problem, it must mean I’m the first person in the world that found some amazing bug on EKS AWS and VPC integrations, right?

Lolno.

If you’re an average Joe and can’t find what you looking on Google, you’re probably:

  1. asking it wrong;
  2. making such a noob mistake that no one’s pathetic enough to come by that problem.

Guess what, it was the second option.

Once again, from AWS Docs, there are two parameters that are DNS relevant here: enableDnsSupport and enableDnsHostnames. Names are pretty intuitive, and it’s impossible to not facepalm after you read them.

Also, on the same page:

By default, both attributes are set to true in a default VPC or a VPC created by the VPC wizard. By default, only the enableDnsSupport attribute is set to true in a VPC created any other way. To check if your VPC is enabled for these attributes, see Viewing and Updating DNS Support for Your VPC.

Guess what, Terraform is “any other way”.

Of course, they say that as well on their docs:

enable_dns_hostnames - (Optional) A boolean flag to enable/disable DNS hostnames in the VPC. Defaults false.

So, fix that and everything should be working.

TODO: as I changed the parameter on my VPC, my hostname was still not DNS-resolvable; I’m not sure how long it takes for it to add the old hostnames to their DNS or if it only works for machines created after you stopped noobing. Since I’m too lazy to research for that (actually, once again, I failed Googl’ing), I opted to destroy my current node and make a new one. Then stuff wen operational again.

Lessons learned

  1. Not only RTFM, but RTFL (L for LOGS): Being “up and running” doesn’t mean it’s working.

  2. If you don’t know what are the default values of the attributes of the stuff you’re creating, you’re doing it wrong: VPCs don’t even have that many attributes, so if you have no idea what the attributes to, maybe you should revisit #1 (RTFM!)

  3. If you’re having problems that no one asked the web before, most likely it’s absolutely trivial and you’re dumb: You might have come across a new savage bug that no one else ever met before, but most likely you don’t.
